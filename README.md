# forge

**Craft Your Digital Future ‚ÄĒ from the terminal.**

The blacksmith was the cornerstone of every civilization. They built the tools that built everything else. The sword, the plow, the cathedral ‚ÄĒ all born at the forge.

Forge is that for the digital age.

A CLI platform where human creativity meets artificial intelligence. Where you don't just store code ‚ÄĒ you craft your entire digital life. Git backups. AI orchestration. Scripture. System management. Creative tools. All connected. All local. All yours.

One forge to shape them.

---

## The Metaphor

Every blacksmith's workshop has its tools. So does Forge.

```
forge heat      Spin up AI agents ‚ÄĒ warm the coals
forge strike    Execute a task across agents
forge quench    Save state, backup, cool down
forge temper    Review, refine, improve quality
forge anneal    Deep work mode ‚ÄĒ slow, methodical, focused
forge alloy     Combine outputs from multiple AI sources
forge cast      Deploy, publish, release into the world
forge grind     Lint, test, sharpen the blade
forge polish    Format, document, make it beautiful
```

## Architecture

```
~/.forge/
‚Ēú‚ĒÄ‚ĒÄ config.toml            Your forge configuration
‚Ēú‚ĒÄ‚ĒÄ vault/                 Encrypted credentials (AES-256)
‚Ēú‚ĒÄ‚ĒÄ db/
‚Ēā   ‚Ēú‚ĒÄ‚ĒÄ forge.db           Core metadata (backups, projects, schedules)
‚Ēā   ‚Ēú‚ĒÄ‚ĒÄ spirit.db          Journal entries & scripture bookmarks
‚Ēā   ‚ĒĒ‚ĒÄ‚ĒÄ agents.db          AI agent state, sessions, history
‚Ēú‚ĒÄ‚ĒÄ archives/              Git backups (zstd compressed, content-deduplicated)
‚Ēú‚ĒÄ‚ĒÄ chunks/                Deduplicated content store (SHA-256 sharded)
‚Ēú‚ĒÄ‚ĒÄ projects/              Project registry and metadata
‚Ēú‚ĒÄ‚ĒÄ prompts/               Versioned prompt library
‚Ēú‚ĒÄ‚ĒÄ themes/                Custom themes (TOML)
‚Ēú‚ĒÄ‚ĒÄ scripts/               Automation hooks and lifecycle events
‚ĒĒ‚ĒÄ‚ĒÄ logs/                  Activity history and audit trail
```

### Archive Format

Each backup produces a `.tar.zst` file containing a bare git clone of the repository. Archives are named `<repo>-<timestamp>.tar.zst`. Metadata (branches, tags, commit count, SHA-256 hash, chunk references) is stored in SQLite for instant querying.

### Content Deduplication

The ChunkStore splits data into 4MB blocks, SHA-256 hashes each one, compresses with zstd, and stores them in a sharded content-addressable layout (`chunks/ab/cdef...zst`). New backups only store chunks they haven't seen before. Across all your projects, this means massive space savings ‚ÄĒ identical dependencies, assets, and boilerplate are stored once.

## Configuration

Default config at `~/.forge/config.toml`:

```toml
[forge]
name = "my-forge"
data_dir = "~/.forge"

[archive]
compression = 3
chunk_size = 4194304          # 4MB

[retention]
keep_daily = 7
keep_weekly = 4
keep_monthly = 12

[theme]
active = "synthwave84"

[agents]
auto_start = false
preferred = "opencode"        # default agent for forge strike
local_model = "llama-swap"

[spirit]
translation = "ESV"
daily_verse = true
sabbath_mode = false
journal_encrypted = true

[bridge]
notifications = ["desktop"]
webhooks = []
```
